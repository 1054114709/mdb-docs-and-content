引导选项卡

引导程序选项卡是用于分隔放置在同一包装程序中但在单独窗格中的内容的组件。一次只能显示一个窗格。

如果您使用导航来提供导航栏，请确保添加一个导航栏。角色=“导航” 到最逻辑的父容器<ul> ，或包装一个<nav>整个导航中的元素。不要将角色添加到<ul>本身，因为这将阻止辅助技术将其宣布为实际列表。

注意： 您可以在中找到更多详细信息 Navs文档

活动

显示新标签时，事件将按以下顺序触发：

  1. hide.bs.tab （在当前活动标签上）
  2. show.bs.tab （在待显示的标签上）
  3. hidden.bs.tab （在上一个活动标签上，与 hide.bs.tab 事件）
  4. shown.bs.tab （在刚刚激活的新标签上，与 show.bs.tab 事件）

如果没有标签处于活动状态，则 hide.bs.tab 和 hidden.bs.tab 事件不会被触发。

事件类型 描述
show.bs.tab 此事件会在标签页显示时触发，但会在新标签页显示之前触发。采用事件目标和 event.relatedTarget 分别定位到活动标签和上一个活动标签（如果有）。
shown.bs.tab 显示标签后，此事件会在标签显示上触发。采用事件目标和 event.relatedTarget 分别定位到活动标签和上一个活动标签（如果有）。
hide.bs.tab 当要显示一个新选项卡时将触发此事件（因此将隐藏先前的活动选项卡）。采用事件目标和 event.relatedTarget 分别定位到当前的活动标签和新的即将投放的标签。
hidden.bs.tab 此事件在显示新标签后触发（因此，先前的活动标签被隐藏）。采用事件目标和 event.relatedTarget 分别定位到上一个活动标签和新的活动标签。
