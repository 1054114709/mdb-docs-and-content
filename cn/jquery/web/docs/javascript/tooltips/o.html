引导工具提示

以下是用于通过CSS和JavaScript使用CSS3添加自定义Bootstrap工具提示的文档和示例，其中CSS3用于动画，数据属性用于本地标题存储。

概述

使用工具提示插件时要了解的事项：

<!-说明->
  • 工具提示依赖于第三方库 Popper.js 用于定位。您必须包括 popper.min.js 在bootstrap.js之前，以便工具提示起作用！
  • 出于性能原因选择使用工具提示，因此你必须自己初始化它们 。
  • 标题长度为零的工具提示永远不会显示。
  • 指定容器：“身体” 以避免在更复杂的组件（例如我们的输入组，按钮组等）中呈现问题。
  • 在隐藏元素上触发工具提示将不起作用。
  • 的工具提示 .disabled 要么残障人士元素必须在包装元素上触发。
  • 从跨越多行的超链接触发时，工具提示将居中。采用空白：nowrap; 在你的<a>避免这种行为。
  • 在从DOM中删除其相应元素之前，必须先隐藏工具提示。

知道了吗？太好了，让我们看看它们是如何工作的。

您可以通过JavaScript触发工具提示：

标记

工具提示所需的标记仅为数据属性和一个标题您希望在HTML元素上有一个工具提示。工具提示的生成标记相当简单，尽管它确实需要位置（默认情况下，设置为最佳通过插件）。

使工具提示对键盘和辅助技术用户有效

您只应向传统上可键盘聚焦和交互的HTML元素添加工具提示（例如链接或表单控件）。尽管任意HTML元素（例如跨度 s）可以通过添加 tabindex =“ 0” 属性，这将为键盘用户在非交互式元素上添加潜在的令人讨厌和令人困惑的制表位。此外，在这种情况下，目前大多数辅助技术都不会公布工具提示。

此外，不要仅仅依靠徘徊作为工具提示的触发器，因为这将使键盘用户无法触发工具提示。

各个工具提示的数据属性

如上所述，可以单独使用数据属性来指定各个工具提示的选项。

方法

异步方法和转换

所有API方法都是异步然后开始过渡 。转换开始后，他们会立即返回给呼叫者，但在结束之前 。此外，在过渡组件将被忽略 。

工具提示选项

将工具提示处理程序附加到元素集合。

.tooltip（&#39;显示&#39;）

显示元素的工具提示。它会在实际显示工具提示之前返回到调用者 （即显示的.bs.tooltip 事件发生）。这被认为是工具提示的“手动”触发。标题长度为零的工具提示永远不会显示。

$（&#39;＃element&#39;）。tooltip（&#39;show&#39;）

.tooltip（&#39;隐藏&#39;）

隐藏元素的工具提示。在工具提示实际上被隐藏之前，它将返回给调用者 （即 hidden.bs.tooltip 事件发生）。这被认为是工具提示的“手动”触发。

> $（&#39;＃element&#39;）。tooltip（&#39;hide&#39;）

.tooltip（&#39;toggle&#39;）

切换元素的工具提示。在工具提示实际显示或隐藏之前，它将返回给调用者 （即显示的.bs.tooltip 要么 hidden.bs.tooltip 事件发生）。这被认为是工具提示的“手动”触发。

$（&#39;＃element&#39;）。tooltip（&#39;toggle&#39;）

.tooltip（&#39;dispose&#39;）

隐藏和销毁元素的工具提示。使用委派的工具提示（使用的选择器选项 ）无法在后代触发元素上单独销毁。

$（&#39;＃element&#39;）。tooltip（&#39;dispose&#39;）

.tooltip（&#39;启用&#39;）

使元素的工具提示能够显示。默认情况下启用工具提示。

$（&#39;＃element&#39;）。tooltip（&#39;enable&#39;）

.tooltip（&#39;禁用&#39;）

删除显示元素工具提示的功能。只有重新启用后，才能显示工具提示。

$（&#39;＃element&#39;）。tooltip（&#39;disable&#39;）

.tooltip（&#39;toggleEnabled&#39;）

切换显示或隐藏元素工具提示的功能。

$（&#39;＃element&#39;）。tooltip（&#39;toggleEnabled&#39;）

.tooltip（&#39;更新&#39;）

更新元素工具提示的位置。

$（&#39;＃element&#39;）。tooltip（&#39;update&#39;）
