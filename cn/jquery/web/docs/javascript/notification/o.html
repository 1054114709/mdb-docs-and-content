可获取性

吐司对您的访问者或用户来说是一个小小的干扰，因此为了帮助那些使用屏幕阅读器和类似辅助技术的人，您应该将吐司包装在咏叹调生活区 。屏幕阅读器会自动宣布对活动区域的更改（例如注入/更新烤面包组件），而无需移动用户的焦点或以其他方式打扰用户。此外，包括 aria-atomic =“ true” 以确保整个烤面包始终以单个（原子）单位宣布，而不是宣布已更改的内容（如果仅更新部分烤面包的内容，或者稍后显示相同的烤面包内容，则可能会导致问题及时）。如果所需的信息对于流程很重要，例如对于表格中的错误列表，请使用警报组件而不是烤面包。

请注意，在生成或更新吐司之前，必须在标记中包含实时区域。如果您同时动态生成两者并将它们注入页面，则辅助技术通常不会宣布它们。

您还需要适应角色和咏叹调级别取决于内容。如果这是一条重要消息（例如错误），请使用角色=“警报” aria-live =“断言” ，否则使用角色=“状态” aria-live =“礼貌” 属性。

当您显示的内容发生更改时，请确保更新延迟超时，以确保人们有足够的时间阅读吐司。

方法

异步方法和转换

所有API方法都是异步然后开始过渡 。转换开始后，他们会立即返回给呼叫者，但在结束之前 。此外，在过渡组件将被忽略 。

$（）。toast（选项）

将Toast处理程序附加到元素集合。

.toast（&#39;显示&#39;）

展现元素的敬酒。在实际显示吐司之前返回呼叫者 （即如图所示事件发生）。您必须手动调用此方法，否则不会显示吐司。

$（&#39;＃element&#39;）。toast（&#39;show&#39;）

.toast（&#39;隐藏&#39;）

隐藏元素的吐司。在烤面包被隐藏之前返回到调用者 （即隐藏的面包吐司事件发生）。您必须手动调用此方法自动隐藏至假 。

$（&#39;＃element&#39;）。toast（&#39;hide&#39;）

.toast（&#39;dispose&#39;）

隐藏元素的吐司。您的敬酒将保留在DOM上，但不再显示。
