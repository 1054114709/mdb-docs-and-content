怎么运行的

在开始使用MDB模态组件之前，请确保阅读以下内容，因为我们的菜单选项最近已更改。

  • 使用HTML，CSS和JavaScript构建模态。 它们位于文档的其他所有内容上，并从中移除滚动， <body> 从而使模式内容滚动。
  • 单击模式“背景”将自动关闭模式。
  • Bootstrap一次仅支持一个模式窗口。 不支持嵌套模式，因为我们认为它们是糟糕的用户体验。
  • 模态使用 position: fixed ，有时可能对其渲染有些特殊。 只要有可能，请将模式HTML放在顶层位置，以避免其他元素的潜在干扰。 .modal 在另一个固定元素中 嵌套时，您可能会遇到问题 。
  • 同样，由于 position: fixed ，在移动设备上使用模态有一些警告。
  • 由于HTML5如何定义其语义， 因此 autofocus HTML属性 在Bootstrap模态中无效。 为了达到相同的效果，请使用一些自定义JavaScript：
  • 的JavaScript 
var myModal = document.getElementById('myModal')
var myInput = document.getElementById('myInput')

myModal.addEventListener('shown.bs.modal', function () {
  myInput.focus()
})

变更动画

该 $modal-fade-transform 变量确定 .modal-dialog 模态淡入动画之前 的变换状态 ，该 $modal-show-transform 变量确定 .modal-dialog 模态淡入动画结束时的 变换 。

例如，如果要放大动画，可以设置 $modal-fade-transform: scale(.8) 。

删除动画

如果模态的高度在打开时发生变化，则应调用 myModal.handleUpdate() 来重新调整模态的位置，以防出现滚动条。

辅助功能

确保将 aria-labelledby="..." 引用模式标题 添加 到 .modal 。 另外，您可以使用 aria-describedby on来 描述模态对话框 .modal 。 请注意，您不需要添加， role="dialog" 因为我们已经通过JavaScript添加了它。
