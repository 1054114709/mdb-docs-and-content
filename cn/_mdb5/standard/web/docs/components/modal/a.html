模态-API

用法

模态插件通过数据属性或JavaScript按需切换隐藏的内容。它还添加.modal-open了，<body>以覆盖默认的滚动行为，并生成一个，.modal-backdrop以提供单击区域，以在模态外部单击时关闭显示的模态。

通过数据属性

无需编写JavaScript即可激活模式。设置 data-toggle="modal"的控制器元件上，像一个按钮，以及一个 data-target="#foo"或href="#foo"，指定特定模态切换。

  • 的HTML
<button type="button" data-toggle="modal" data-target="#myModal">
  Launch modal
</button>

通过JavaScript

用一行JavaScript创建一个模态：

  • 的JavaScript
var myModal = new mdb.Modal(document.getElementById('myModal'), options)

选件

可以通过数据属性或JavaScript传递选项。对于数据属性，将选项名称附加到中data-，如中所示data-backdrop=""。

名称 类型 默认 描述
backdrop 布尔值或字符串 'static' true 包括一个模式背景元素。或者，指定 static不关闭点击模式的背景。
keyboard 布尔值 true 按下退出键时关闭模式
focus 布尔值 true 初始化时将重点放在模态上。
show 布尔值 true 初始化时显示模态。

方法

异步方法和过渡：
所有API方法都是异步的，并开始过渡。一旦转换开始但在转换结束之前，它们将返回给呼叫者 。此外，将忽略对过渡组件的方法调用 。

传递选项

将您的内容激活为模式。接受可选选项 object。

方法 描述
dispose 销毁元素的模态。
getInstance 允许您获取与DOM元素关联的模式实例的静态方法
handleUpdate 如果模态的高度在打开时发生变化，则手动重新调整模态的位置（即，如果出现滚动条）。
hide 手动隐藏模式。 在实际隐藏模式 之前（即hidden.bs.modal事件发生之前）返回调用方。
show 手动打开模式。 在实际显示模式 之前（即shown.bs.modal事件发生之前）返回调用方。
toggle 手动切换模态。 在实际显示或隐藏模式之前 （即在shown.bs.modalorhidden.bs.modal事件发生之前）返回调用方。
  • javascript
var myModalEl = document.getElementById('myModal')
var modal = new mdb.Modal(myModalEl)
modal.show()

大事记

Bootstrap的模态类公开了一些事件，可以挂接到模态功能上。所有模态事件都向模态本身触发（即 <div class="modal">）。

事件类型 描述
show.bs.modal show调用实例方法 时，将立即触发此事件。如果是由单击引起的，则clicked元素可用作relatedTarget事件的 属性。
shown.bs.modal 当模式对用户可见时将触发此事件（将等待CSS转换完成）​​。如果是由单击引起的，则clicked元素可用作relatedTarget事件的 属性。
hide.bs.modal hide调用实例方法后， 立即触发此事件。
hide.bs.modal 当模态完成向用户隐藏时将触发此事件（将等待CSS转换完成）​​。
hide.bs.modal 当显示模态时显示此事件，其背景为， static并在模态外部单击或使用键盘选项或data-keyboard将设置为进行退出键false。
  • 的JavaScript
var myModalEl = document.getElementById('myModal')
myModalEl.addEventListener('hidden.bs.modal', function (e) {
  // do something...
})
