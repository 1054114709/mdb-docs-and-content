引导验证

通过HTML5表单验证为您的用户提供有价值的可行反馈– 在我们所有受支持的浏览器中可用 。从浏览器的默认验证反馈中进行选择，或使用我们的内置类和入门JavaScript实施自定义消息。

警告

当前，我们建议您使用自定义验证样式，因为本机浏览器的默认验证消息不会在所有浏览器（最著名的是台式机和移动设备上的Chrome）中始终暴露于辅助技术。

工作原理

这是表单验证与Bootstrap一起工作的方式：

  • HTML表单验证是通过CSS的两个伪类应用的， ：无效和 ：有效 。适用于<input> ，<select>和<textarea>元素。
  • 引导范围 ：无效和 ：有效父样式已验证类，通常应用于<form> 。否则，任何没有值的必填字段在页面加载时都将显示为无效。这样，您可以选择何时激活它们（通常在尝试提交表单之后）。
  • 要重置表单的外观（例如，使用AJAX动态提交表单），请删除已验证来自的类<form>提交后再次提交。
  • 作为后备， 。是无效的和 。已验证可以使用类代替伪类服务器端验证 。他们不需要已验证家长班。
  • 由于CSS运作方式的限制，我们目前无法将样式套用至<label>无需自定义JavaScript即可在DOM中的表单控件之前出现。
  • 所有现代浏览器均支持约束验证API ，一系列用于验证表单控件的JavaScript方法。
  • 反馈消息可以利用浏览器默认值 （因浏览器而异，并且无法通过CSS样式化）或带有其他HTML和CSS的自定义反馈样式。
  • 您可以通过以下方式提供自定义有效性消息： setCustomValidity 在JavaScript中。

考虑到这一点，请考虑以下有关我们的自定义表单验证样式，可选的服务器端类和浏览器默认值的演示。
