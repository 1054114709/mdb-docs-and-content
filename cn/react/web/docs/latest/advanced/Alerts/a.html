...但是您可以定义自己的过渡！烤面包依靠反应过渡基团对于进入和退出过渡，因此使用它构建的任何过渡都可以！让我们来看看动画是如何定义的放大来自的动画 animate.css 举个例子。

/ * style.css * / @keyframes zoomIn {from {opacity：0;转换：scale3d（。3.3.3）; } 50％{不透明度：1； }} .zoomIn {animation-name：zoomIn; } @keyframes zoomOut {from {opacity：1; } 50％{不透明度：0；转换：scale3d（。3.3.3）; }至{opacity：0; }} .zoomOut {animation-name：zoomOut; } / *不需要cssTransition帮助程序* / .animate {animation-duration：800ms; }

进行自己的过渡的最简单方法是使用 cssTransition 帮助器，这使我们可以跳过所有反应过渡基团相关的麻烦。这里唯一需要的是输入和出口类名。默认情况下，过渡时间设置为750ms，但可以覆盖：

从&#39;react&#39;导入React，{组件};从“ mdbreact”导入{toast，cssTransition}；导入&#39;./style.css&#39;; const Zoom = cssTransition（{输入：&#39;zoomIn&#39;，退出：&#39;zoomOut&#39;，//默认为750ms，可以省略duration = 750，//具有不同的退出/进入时间，传入数组}）；类App扩展了组件{notify =（）=> {toast（“ ZoomIn and ZoomOut”，{transition：Zoom，autoClose：5000}）; }; render（）{返回 ; }}
