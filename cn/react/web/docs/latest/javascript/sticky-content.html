React Sticky（Bootstrap词缀） MDB Pro组件

React Sticky是一个特殊的组件，它允许元素固定（锁定）到页面上的某个区域。创建它是为了允许黏性行为 ，而CSS属性（ 位置：粘性 ）仅得到部分支持。它提供了一种有趣的自定义导航组件，社交媒体按钮，小部件和用户界面其他动态部分的方式。

使用

步骤1： 导入并应用<StickyContener>零件。它将充当包装器和盒子，通过它们我们将塑造<Sticky> 。当然，我们可以根据需要调整样式并填充组件。它支持所有有效<div>道具。

<StickyContainer style={{ background: "#fff", height: '465px' }}>
        ...
</StickyContainer>

刚开始时可能会造成混淆，但是使用这种解决方案可以更好地控制黏性发生。如下所示，<Sticky>可能会包含一些方便的对象属性。

不动产名称 类型 含义
isSticky 布尔值 确定元素是否为黏由于当前事件
wasSticky 布尔值 用于确定元素是否为黏之前事件。
distanceFromTop 数字 从顶部开始的像素数<Sticky>到最近<StickyContainer> &#39;停
distanceFromBottom 数字 从顶部开始的像素数<Sticky>到最近<StickyContainer>的底部
计算高度 数字 此函数返回的元素的高度
风范 物体 可修改的样式属性，可以选择传递给此函数返回的元素

最后，<Sticky>本身可能会带有一些道具：

道具名称和默认值 含义
相对的 （默认： 假 ） 如果将相对设置为true <Sticky>元素将在溢出中呈现<StickyContainer/> （例如样式= {{ overflowY: 'auto' }} ），而您想要<Sticky>仅对该容器内的事件做出反应的行为。当在相对的模式，窗口事件不会触发粘性状态更改。仅在最近的位置滚动<StickyContainer>可以触发粘性状态更改。
topOffset （默认： 0 ） 当元素的顶部为top时，将触发粘性状态<StickyContainer> 。正数给人的印象是懒惰的粘滞状态，而负数则更倾向于粘贴。 topOffset = {80} 一旦元素的顶部距离顶部至少大于或等于80像素，将导致该元素变为粘性<StickyContainer> 。
底部偏移 （默认： 0 ） 当元素的底部是 bottomOffset 最接近底部的像素<StickyContainer> 。
disableCompensation （默认： 假 ）

调成真正如果你不想要你的<Sticky>将填充应用于隐藏的占位符<div>纠正“跳动”，因为附件从位置：固定然后回来。
disableHardwareAcceleration （默认： 假 ） 何时设置为真正 ，<Sticky>元素将不使用硬件加速（例如转换：translateZ（0） ）。不建议使用此设置，因为它会对移动体验产生负面影响，通常可以通过改进DOM的结构来避免此设置。
